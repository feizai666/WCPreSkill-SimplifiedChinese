# 世界杯赛事比分预测（World Cup Match Predictor）

为指定日期的世界杯比赛生成**比分预测**与**投注参考**。流程分五步：赛后复盘校准 → 确定赛程 → 多维信息收集 → 综合分析建模 → 输出预测报告（含赔率对比与下注建议）。

> ⚠️ 投注建议仅针对世界杯比赛，且仅供参考。理性娱乐，所有预测均存在不确定性，不构成任何盈利保证。

---

## 触发与输入

- **默认行为**：预测「北京时间明天」的所有世界杯比赛。
- **可指定日期**：用户给出具体日期（如"6月20日"、"后天"）则预测当日比赛。
- **可指定单场**：用户只问某两队对阵，则只分析该场。
- **定时调用**：每天下午由定时任务触发时，等同"预测明天所有比赛"。

开始前，用 `get_current_time`（builtin MCP）确认"今天/明天"对应的真实日期，避免日期算错。

---

## 第零步：赛后复盘与模型校准（必须先做）

每次生成新报告前，先复盘本技能近期已经生成过的预测报告，尤其是 `reports/worldcup_*_predictions.json` 中目标日期早于当前日期的比赛。

1. 读取历史报告里的每场预测：
   - 首选比分、次选比分、胜平负概率、总进球预测、让球/总进球/波胆建议。
2. 联网核对真实赛果与关键比赛过程：
   - 最终比分、半场比分、红牌/点球/早球、关键伤退、轮换、天气或场地显著影响。
   - 若无法查到完整过程，至少核对最终比分，并明确过程信息不足。
3. 做结构化误差归因：
   - **方向误差**：胜平负是否判断错。
   - **比分误差**：首选/次选是否命中，净胜球差和总进球差偏大还是偏小。
   - **赔率误差**：让球盘、总进球最低档、波胆市场是否比自己判断更接近真实结果。
   - **思维误差**：是否套用了固定模板（例如"强队打弱队=3-0"、"均势队=1-0/1-1"），是否低估大比分、冷门、早球导致的开放节奏、末轮/第二轮净胜球动机。
4. 把复盘结论写进新报告的汇总部分，作为"今日校准"：
   - 明确本次会提高或降低哪些因素权重（如早球后比赛开放、深让盘被打穿概率、弱队反击进球概率）。
   - 如果连续预测出现同类比分模板，必须主动打破模板，给出更宽的比分分布。
5. 复盘只用于校准，不要为了"显得不一样"而故意逆市场或制造冷门；所有调整仍需回到真实信息与赔率结构。

## 第一步：确定赛程

1. 用 `web_search` 搜索目标日期的世界杯赛程，关键词示例：`世界杯 {日期} 赛程 对阵`、`World Cup fixtures {date}`。
2. 对每场比赛提取：
   - 对阵双方、所属小组
   - **开球时间（北京时间 + 当地时间）** —— 当地时间用于评估气候/时差/球员适应
   - 比赛城市与球场
3. 列出当天全部比赛，逐场进入后续分析。

## 第二步：多维信息收集（每场比赛都要做）

这是预测质量的核心。**必须实际联网收集**，不能凭空臆测。对阵双方各自收集以下维度，建议并行发起多次 `web_search`，再对关键结论用 `web_fetch` 核实。

### A. 近期状态（近半年所有国际赛事）
- 搜集两队近半年**全部国际赛事**成绩（世界杯、预选赛、热身赛、洲际赛事、欧国联等）。
- 关注：胜负走势、进失球数、连胜/连败、对强队/弱队的不同表现。
- 判断"势头"：是上升期还是低迷期。

### B. 本届世界杯进程与状态
- 该队本届已踢的比赛结果与表现（控球、射门、防守漏洞）。
- 体能消耗、是否经历加时、连续作战间隔天数。

### C. 历史交锋战绩（H2H）
- 两队过往交锋记录，尤其近 5–10 次。
- 大赛中的交锋（世界杯/洲际杯往往比热身赛更有参考价值）。

### D. 打法风格与战术克制
- 双方阵型与风格（控球型 / 防反型 / 高位逼抢 / 定位球强队等）。
- 风格相性：谁的打法克制谁，中场与边路对位。

### E. 球员状态与阵容
- 核心球员状态、近期是否有伤、关键射手/中场是否健康。
- 门将状态、后防稳定性。

### F. 停赛与黄牌风险（关键且易被忽略）
- **本场确定停赛**的球员（红牌或累计黄牌已达停赛）。
- **黄牌累计风险**：小组赛阶段有球员处于"再吃 1 黄即停赛"，可能影响其拼抢强度与表现。
- 伤病停赛、临时退队情况。

### G. 小组出线形势（小组赛阶段尤为重要）
- 当前小组积分榜（同组其他队可能已先踢）。
- 各队出线需求：是必须赢、可以保平、还是已出线可能轮换。
- "默契球"风险：末轮双方都满意某比分时，可能踢得保守。

### H. 天气、场地与外部因素
- 主客场 / 中立场，当地球迷氛围与到场支持。
- **未来天气预报**：开球时温度、湿度、降雨概率、风速、体感温度；室内/可开合顶棚球场要确认顶棚状态或至少注明不确定。
- 气候（高温、高湿、高原、雨战、强风）、时差、长途奔波。
- 草皮、场地尺寸、人工/天然草、封闭球场或开放球场对传控、长传、定位球和体能的影响。

### I. 主裁判与出牌尺度（淘汰赛尤为关键）
- 查询本场主裁判与 VAR/第四官员（若官方尚未公布，必须标注“未公布，按可能人选/赛事平均尺度估计”，不得臆造姓名）。
- 收集主裁判过去国际赛事、洲际赛事或本届世界杯的执法数据：
  - 场均黄牌、红牌、点球、犯规尺度、是否常早牌控场。
  - 是否偏向保护比赛流畅，还是更愿意用牌压制身体对抗。
  - 典型例子可用作校准，例如马宁这类整体出牌较多的裁判。
- 结合两队打法、逼抢强度、边路一对一、防守转换犯规、淘汰赛压力，给出：
  - **预计总黄牌数区间**（如 3-5 张 / 5-7 张）
  - 红牌或点球尾部风险
  - 哪一队更可能吃牌，以及原因

### J. 主教练、临场调整与 26 人后手
- 分析双方主教练的战术偏好、保守/激进程度、临场换人质量、淘汰赛经验和短板。
- 不只看首发 11 人，要把 26 人名单视为一整个 team：
  - 替补席是否有改变节奏的边锋、中锋、定位球手、后腰、防守型换人。
  - 领先后能否守住，落后后是否有 Plan B。
  - 主教练是否存在固执、慢换人、不会保护领先、过度轮换等已知风险。
- 对类似 Bielsa、Roberto Martínez 这类风格鲜明或临场争议较多的教练，要把“风格在世界杯淘汰赛能否兑现”单独作为变量，而不是只看纸面阵容。

### K. 淘汰赛状态与加时/点球
- 淘汰赛阶段必须同时输出两层判断：
  - **常规时间预测**：90 分钟比分、胜平负概率、总进球倾向。
  - **全场/晋级预测**：若常规时间打平，推测加时赛是否进球、是否进入点球大战、最终晋级队。
- 评估加时赛和点球大战概率：
  - 双方常规时间保守程度、替补深度、体能消耗、门将扑点能力、点球手储备。
  - 教练是否会在 70 分钟后主动求稳，还是换攻击手争取 90 分钟解决。
  - 若预测进入点球大战，要给出点球大战胜率倾向和依据。

### L. 人名所属队伍标注（必须）
- 报告中提到球员、主教练、助理教练、主裁判等具体人名时，必须在人名后加三字母国家/队伍简称，方便读者区分身份归属。
- 球员、主教练、助教默认标注其**本场所属球队**，例如 `Davies（CAN）`、`Hugo Broos（RSA）`、`Marsch（CAN）`。
- 主裁判、VAR、第四官员默认标注其**国籍**，例如 `João Pinheiro（POR）`、`马宁（CHN）`。
- 若主教练国籍和执教队伍不同，且该差异对分析有意义，可写成 `Roberto Martínez（POR 主帅/ESP）`；但普通比赛分析中至少要保留执教队伍简称。
- 一张卡片内第一次出现人名时必须标注；若同一段同时提到双方多名球员/教练，重复出现也要尽量标注，避免用户需要凭记忆判断归属。
- 在 `key_factors`、`coach_tactics`、`bench_depth`、`referee`、`card_prediction`、复盘卡正文和投注建议中都适用本规则。

> 收集时优先权威来源（FIFA、各国足协、主流体育媒体、专业数据站）。把每队的关键结论汇总成简短要点，供建模使用。

## 第三步：综合分析与比分建模

参考 `references/prediction-framework.md` 的权重框架，对收集到的信息做结构化推理：

1. 为每队估算一个"综合实力/状态分"，列出加分项与减分项。
2. 结合 H2H 与风格克制做调整。
3. 叠加情境因素（出线需求、停赛、气候等）。
4. 先列出 4–6 个候选比分，并标注它们对应的比赛脚本：
   - 强队早进球后继续打开（如 4-0、4-1、5-1）
   - 强队久攻不下但控制局面（如 1-0、2-0）
   - 弱队先进球或偷到反击（如 2-1、3-1、1-1）
   - 红牌/点球/天气/末轮净胜球需求造成的高方差比分
5. 单独估算纪律与判罚变量：
   - 预计总黄牌数、红牌/点球概率、判罚尺度会让比赛更碎还是更流畅。
   - 若裁判未公布，不得编造姓名；用赛事平均尺度 + 两队风格估计，并明确不确定。
6. 淘汰赛阶段单独估算常规时间与晋级路径：
   - 常规时间比分和概率。
   - 加时赛概率、点球大战概率、最终晋级概率。
   - 不能把“常规时间平局”和“全场胜负”混为一谈。
7. 检查是否陷入模板化：
   - 不得仅凭"强弱明显"默认 3-0。
   - 不得仅凭"实力接近"默认 1-0 或 1-1。
   - 如果最终选择常见比分，必须说明为什么它比高比分/冷门脚本更合理。
8. 得出：
   - **预测比分**（小组赛为全场 90 分钟；淘汰赛默认写常规时间比分，必要时附全场晋级判断）
   - **总进球数预测**（预计进球总数，及大/小球倾向，如"预计 2-3 球，偏大 2.5"）
   - **胜平负倾向** 与 **大致概率分布**（如 主胜 50% / 平 28% / 客胜 22%）
   - **出牌数预测**（黄牌区间、红牌/点球风险、执法尺度判断）
   - **教练与后手判断**（临场调整、替补深度、领先/落后脚本）
   - **淘汰赛晋级判断**（若适用：常规时间、加时、点球、晋级队）
   - **置信度**（高/中/低）与**关键不确定点**

务必写出**推理依据**，而非只给结论。

## 第四步：赔率抓取与投注建议（仅限世界杯比赛）

**只使用竞彩（中国体育彩票）官方赔率**，不再抓取北单。需抓取竞彩的 **4 类玩法**，并据此分析比分、总进球与下注方向。

### 赔率来源（竞彩官方，4 个独立玩法页面）
| 玩法 | URL |
|------|-----|
| 胜平负 / 让球胜平负 | `https://www.sporttery.cn/jc/jsq/zqspf/` |
| 比分（波胆） | `https://www.sporttery.cn/jc/jsq/zqbf/` |
| 总进球数 | `https://www.sporttery.cn/jc/jsq/zqzjq/` |
| 半全场（可选） | `https://www.sporttery.cn/jc/jsq/zqbqc/` |

> ⚠️ 竞彩正确入口是 `/jc/jsq/...`（不是 `/ic/isq/...`，旧路径已失效跳错误页）。若 URL 失效，从胜平负页导航栏链接里重新解析各玩法路径。

### ⚠️ 抓取方式（重要）
这些页面都是 **JavaScript 动态渲染**的，直接 `web_fetch` 只能拿到空表头或错误页。**必须用 Playwright**：

1. `browser_navigate` 打开玩法页 → `browser_wait_for(text="世界杯")` 等渲染。
2. 用 `browser_evaluate` 读 `document.body.innerText`，按 `06-XX` 日期 + 球队名定位当天世界杯场次。
3. **胜平负页**：提取每场「不让球」胜/平/负 + 「让球胜平负」让球数与胜/平/负赔率 + 大众支持率。
4. **总进球数页**：提取每场 0/1/2/3/4/5/6/7+ 球各档赔率，找**赔率最低档**（市场预期进球数）。
5. **比分页**：每场比分块默认**折叠**，需先点击展开按钮（`span.folderTd`，文本为 `+`）再读取；取**赔率最低的几个比分**（市场最看好的波胆）。
6. 抓取失败或当天无世界杯场次，如实说明并跳过，**不要编造赔率**。

### 价值分析与下注建议（结合 4 类玩法）
1. 把第三步估算的胜平负概率换算成"合理赔率"（≈ 1 / 概率），与竞彩实际赔率对比找价值（实际 > 合理 → 有价值）。
2. **胜平负 / 让球胜平负**：给出推荐方向；当不让球赔率过低（如 < 1.3）时，评估让球盘是否更有价值。
3. **总进球数**：结合市场最低档与自己对两队攻防的判断，给出"大/小球"倾向（如大 2.5 / 小 2.5），并指出市场预期的进球档。
4. **比分（波胆）**：把自己的预测比分与市场最看好的几个比分对照，给出"预测比分 + 高赔博胆比分"建议。
5. 每个推荐标注**风险等级**（稳胆 / 博胆 / 高风险），可附单关/串关思路。
6. **每场都附风险提示**，报告开头统一声明：投注有风险，仅供参考，理性娱乐。

## 输出格式

为每场比赛输出结构化结果，并在最后生成一份汇总。**面向用户的报告产物统一为 PNG 格式**，保存到目标日期目录，并用 `declare_artifact` 声明。

PNG 输出规则：
- 目录：`reports/YYYY-MM-DD/`，其中 `YYYY-MM-DD` 是目标比赛日的北京时间日期（例如明天是 2026-06-25，则输出到 `reports/2026-06-25/`）。
- 数量：每次生成先输出 1 张 `00_赛后复盘_今日校准.png`，再每场比赛 1 张 PNG；当天 6 场比赛就输出 7 张 PNG，不额外生成汇总 PNG。
- 命名：`00_赛后复盘_今日校准.png`、`01_主队_vs_客队.png`、`02_主队_vs_客队.png` 等；比赛卡按开球时间排序。
- 00 号卡内容：对前一比赛日真实赛果、比赛过程和历史预测逐场复盘；既指出预测错在哪里，也保留预测对的推理链；小组赛末轮必须重点检查积分、同分胜负关系、净胜球、进球数、红黄牌，以及第三名出线门槛。
- 比赛卡内容：只包含单场比赛卡片信息：对阵信息、预测比分、总进球预测、胜平负概率、关键分析、竞彩赔率与投注建议；赛后复盘/今日校准集中绘制在 00 号卡，不重复绘制到每张比赛卡片。
- 淘汰赛卡片：`预测比分`默认指**常规时间比分**；卡片需额外显示“淘汰赛预测 / 全场晋级”区块，说明是否可能加时、是否可能点球、最终晋级队。
- 扩展分析区块：若 JSON 中包含 `weather`、`referee`、`card_prediction`、`coach_tactics`、`bench_depth`、`knockout` 等字段，PNG 卡片必须显示这些内容，避免只在内部推理但不交付给用户。
- 人名标注：球员/教练/裁判等人名后必须加三字母国家或队伍简称，如 `David（CAN）`、`Broos（RSA）`、`João Pinheiro（POR）`；淘汰赛卡片尤其要给替补后手和主教练标注所属队伍。
- 清晰度：PNG 使用高密度画布导出并写入 300 DPI 元数据；中文字体使用宋体，英文、数字和赔率使用 Times New Roman；中英文与数字混排必须按共同基线对齐，避免数字或英文相对中文上下错位。
- 结构化 JSON 可保留为中间数据源，但不要再把 HTML 作为报告交付格式。

每场比赛包含：

| 字段 | 说明 |
|------|------|
| 对阵 & 小组 | 主队 vs 客队，所属小组 |
| 开球时间 | 北京时间 / 当地时间 |
| 预测比分 | 最可能比分（可附次选） |
| 胜平负倾向 | 主胜/平/客胜 + 概率分布 |
| 总进球预测 | 预计进球数 / 大小球倾向 |
| 天气场地 | 开球时天气、湿度/降雨/风速、顶棚或场地因素 |
| 主裁判与出牌 | 主裁判姓名（若已公布）、执法尺度、预计黄牌区间、红牌/点球尾部风险 |
| 教练与替补后手 | 主教练风格、临场换人、替补深度、领先/落后 Plan B |
| 淘汰赛预测 | 常规时间比分、加时/点球概率、最终晋级队（淘汰赛必填） |
| 关键分析 | 状态、H2H、风格、停赛、出线形势等要点 |
| 置信度 | 高/中/低 + 主要不确定点 |
| 竞彩赔率 | 胜平负 + 让球胜平负 + 总进球数(最低档) + 比分(最看好) |
| 投注建议 | 方向、玩法（胜平负/让球/总进球/比分）、风险等级（仅世界杯） |

汇总部分：当天所有比赛一览表 + 综合下注建议（如"今日推荐 2 场，1 稳胆 1 博胆"）。

汇总部分还必须包含：
- **赛后复盘**：近期历史预测的命中与误差摘要。
- **今日校准**：本次预测如何根据误差避免模板化，尤其说明哪些比赛存在大比分或冷门尾部风险。

## 注意事项

- **必须真实联网**收集数据，信息不足时如实说明，不要臆造成绩/赔率/伤停。
- 赔率页面是动态的，**坚持用 Playwright**，不要因为 `web_fetch` 失败就放弃。
- **只用竞彩官方赔率**（`/jc/jsq/...`），不再使用北单；比分玩法默认折叠需点开 `span.folderTd`。
- 区分「不让球胜平负」与「让球胜平负」，避免赔率误读。
- 总进球数取赔率最低档作为市场预期；比分取赔率最低的几个作为市场最看好波胆。
- 不要让比分预测过度趋同；每场都要评估大比分、冷门、弱队进球和早球打开局面的概率。
- 小组赛后期务必查清积分榜与出线形势，这对预测和"默契球"判断影响极大。
- 淘汰赛必须区分常规时间和最终晋级；常规时间平局不等于没有胜负判断。
- 主裁判未公布时，不得编造裁判姓名；可以写“裁判未公布，按赛事平均尺度估计”，并降低出牌判断置信度。
- 主教练和替补后手是淘汰赛重要变量；不要只看首发纸面实力。
- 球员、教练、裁判等人名后必须带国家/队伍三字母简称；不要默认用户记得每个人属于哪支队。
- 时间一律标注北京时间与当地时间，避免混淆。
- 投注建议**仅针对世界杯**，且附风险提示。
