# 世界杯赛事比分预测（World Cup Match Predictor）

为指定日期的世界杯比赛生成**比分预测**与**投注参考**。流程分四步：确定赛程 → 多维信息收集 → 综合分析建模 → 输出预测报告（含赔率对比与下注建议）。

> ⚠️ 投注建议仅针对世界杯比赛，且仅供参考。理性娱乐，所有预测均存在不确定性，不构成任何盈利保证。

---

## 触发与输入

- **默认行为**：预测「北京时间明天」的所有世界杯比赛。
- **可指定日期**：用户给出具体日期（如"6月20日"、"后天"）则预测当日比赛。
- **可指定单场**：用户只问某两队对阵，则只分析该场。
- **定时调用**：每天下午由定时任务触发时，等同"预测明天所有比赛"。

开始前，用 `get_current_time`（builtin MCP）确认"今天/明天"对应的真实日期，避免日期算错。

---

## 第一步：确定赛程

1. 用 `web_search` 搜索目标日期的世界杯赛程，关键词示例：`世界杯 {日期} 赛程 对阵`、`World Cup fixtures {date}`。
2. 对每场比赛提取：
   - 对阵双方、所属小组
   - **开球时间（北京时间 + 当地时间）** —— 当地时间用于评估气候/时差/球员适应
   - 比赛城市与球场
3. 列出当天全部比赛，逐场进入后续分析。

---

## 第二步：多维信息收集（每场比赛都要做）

这是预测质量的核心。**必须实际联网收集**，不能凭空臆测。对阵双方各自收集以下维度，建议并行发起多次 `web_search`，再对关键结论用 `web_fetch` 核实。

### A. 近期状态（近半年所有国际赛事）
- 搜集两队近半年**全部国际赛事**成绩（世界杯、预选赛、热身赛、洲际赛事、欧国联等）。
- 关注：胜负走势、进失球数、连胜/连败、对强队/弱队的不同表现。
- 判断"势头"：是上升期还是低迷期。

### B. 本届世界杯进程与状态
- 该队本届已踢的比赛结果与表现（控球、射门、防守漏洞）。
- 体能消耗、是否经历加时、连续作战间隔天数。

### C. 历史交锋战绩（H2H）
- 两队过往交锋记录，尤其近 5–10 次。
- 大赛中的交锋（世界杯/洲际杯往往比热身赛更有参考价值）。

### D. 打法风格与战术克制
- 双方阵型与风格（控球型 / 防反型 / 高位逼抢 / 定位球强队等）。
- 风格相性：谁的打法克制谁，中场与边路对位。

### E. 球员状态与阵容
- 核心球员状态、近期是否有伤、关键射手/中场是否健康。
- 门将状态、后防稳定性。

### F. 停赛与黄牌风险（关键且易被忽略）
- **本场确定停赛**的球员（红牌或累计黄牌已达停赛）。
- **黄牌累计风险**：小组赛阶段有球员处于"再吃 1 黄即停赛"，可能影响其拼抢强度与表现。
- 伤病停赛、临时退队情况。

### G. 小组出线形势（小组赛阶段尤为重要）
- 当前小组积分榜（同组其他队可能已先踢）。
- 各队出线需求：是必须赢、可以保平、还是已出线可能轮换。
- "默契球"风险：末轮双方都满意某比分时，可能踢得保守。

### H. 外部因素
- 主客场 / 中立场，当地球迷氛围与到场支持。
- 气候（高温、高原、湿度）、时差、长途奔波。
- 草皮、天气预报（雨战影响技术型球队）。
- 裁判尺度（影响点球/红牌概率，可选）。

> 收集时优先权威来源（FIFA、各国足协、主流体育媒体、专业数据站）。把每队的关键结论汇总成简短要点，供建模使用。

---

## 第三步：综合分析与比分建模

参考 `references/prediction-framework.md` 的权重框架，对收集到的信息做结构化推理：

1. 为每队估算一个"综合实力/状态分"，列出加分项与减分项。
2. 结合 H2H 与风格克制做调整。
3. 叠加情境因素（出线需求、停赛、气候等）。
4. 得出：
   - **预测比分**（给最可能的 1 个，必要时附 1 个次可能比分）
   - **胜平负倾向** 与 **大致概率分布**（如 主胜 50% / 平 28% / 客胜 22%）
   - **置信度**（高/中/低）与**关键不确定点**

务必写出**推理依据**，而非只给结论。

---

## 第四步：赔率抓取与投注建议（仅限世界杯比赛）

收集北单与竞彩的实时胜平负赔率，与自己的概率判断对比，找"价值"。

### 赔率来源
- **北京单场（北单）胜平负**：`https://www.bjlot.com.cn/ssm/dc200_spf.shtml`
- **竞彩（中国体育彩票）胜平负**：`https://www.sporttery.cn/ic/isq/zqspf/`

### ⚠️ 抓取方式（重要）
这两个页面都是 **JavaScript 动态渲染**的，直接 `web_fetch` 只能拿到空表头或 404。**必须用 Playwright** 浏览器自动化加载后再读取：

1. 先 `mcp_get_tool_description` 获取 `browser_navigate` 等工具的参数。
2. `browser_navigate` 打开赔率 URL。
3. `browser_wait_for` 等待赔率表格渲染完成（等待赔率数字/球队名出现）。
4. 用 `browser_snapshot` 或 `browser_evaluate` 提取赔率表格的 DOM 数据（优先 DOM 文本，不要截图）。
5. 从表格中筛出**世界杯**相关场次（按赛事名/球队名匹配当天要预测的比赛）。
6. 提取每场的 **胜 / 平 / 负** 赔率（北单注意"让球"与"不让球"区分；竞彩注意"胜平负"与"让球胜平负"区分，本 skill 默认看**不让球/标准胜平负**，如有让球盘可一并记录）。

若某网站当天无对应世界杯场次或抓取失败，说明情况并跳过该来源，不要编造赔率。

### 价值分析与下注建议
1. 把自己第三步估算的胜平负概率换算成"合理赔率"（合理赔率 ≈ 1 / 概率）。
2. 与北单、竞彩的实际赔率对比：
   - **实际赔率 > 合理赔率** → 存在价值（庄家可能低估），可作为推荐方向。
   - **实际赔率 < 合理赔率** → 价值不足，谨慎。
3. 给出建议时包含：
   - 推荐的投注方向（主胜/平/客胜，或注明"无明显价值，建议观望"）
   - 北单 vs 竞彩 哪边赔率更优（同一方向选赔率更高的平台）
   - 风险等级（稳胆 / 博胆 / 高风险）
   - 可选玩法提示（单关 / 串关组合思路）
4. **每场都附风险提示**，并在报告开头统一声明：投注有风险，仅供参考，理性娱乐。

---

## 输出格式

为每场比赛输出结构化结果，并在最后生成一份汇总。**当天有多场比赛或用户需要美观报告时，生成自包含 HTML 报告**（内联 CSS，无外部依赖），保存到输出目录，并用 `declare_artifact` 声明。

每场比赛包含：

| 字段 | 说明 |
|------|------|
| 对阵 & 小组 | 主队 vs 客队，所属小组 |
| 开球时间 | 北京时间 / 当地时间 |
| 预测比分 | 最可能比分（可附次选） |
| 胜平负倾向 | 主胜/平/客胜 + 概率分布 |
| 关键分析 | 状态、H2H、风格、停赛、出线形势等要点 |
| 置信度 | 高/中/低 + 主要不确定点 |
| 赔率对比 | 北单/竞彩 胜平负赔率 |
| 投注建议 | 方向、平台优选、风险等级（仅世界杯） |

汇总部分：当天所有比赛一览表 + 综合下注建议（如"今日推荐 2 场，1 稳胆 1 博胆"）。

## 工作流程检查清单

1. ✅ `get_current_time` 确认目标日期
2. ✅ 搜索并列出当天全部世界杯比赛
3. ✅ 每场逐维度联网收集信息（第二步 A–H）
4. ✅ 结合框架建模，得出预测比分与概率
5. ✅ Playwright 抓取北单 + 竞彩赔率（动态页面）
6. ✅ 价值对比，生成投注建议（仅世界杯）
7. ✅ 输出结构化报告（多场则生成 HTML 并 declare_artifact）
8. ✅ 全程声明：仅供参考，理性投注

---

## 注意事项

- **必须真实联网**收集数据，信息不足时如实说明，不要臆造成绩/赔率/伤停。
- 赔率页面是动态的，**坚持用 Playwright**，不要因为 `web_fetch` 失败就放弃。
- 区分让球盘与不让球盘，避免赔率误读。
- 小组赛后期务必查清积分榜与出线形势，这对预测和"默契球"判断影响极大。
- 时间一律标注北京时间与当地时间，避免混淆。
- 投注建议**仅针对世界杯**，且附风险提示。
